# 评估指南

用这份指南评估 Redbit 时，不只看它能不能生成媒体，更要看它能不能支撑真实工作流：创意路径、供应商路由、素材复用、Agent 边界和团队交接。

## 谁应该阅读本文

| 读者 | 适合用本文解决什么问题 |
| --- | --- |
| 潜在用户 | 判断 Redbit 是否适合自己的创意工作类型 |
| 团队 lead | 把 Redbit 与直接供应商控制台、聊天工具或内部工具做对比 |
| 工程或创意执行者 | 设计一个不暴露过多凭证和源素材的小型评估流程 |

## 评估前

如果产品术语还不熟，先读 [核心概念与术语表](./concepts.mdx)。Redbit 负责协调卡片、素材、供应商和可选本地集成；模型额度、供应商在线状态、模型政策、计费和输出质量仍由供应商决定。

## 评估结论应该回答什么

一次有效评估结束后，你应该能回答：

| 问题 | 有效证据 |
| --- | --- |
| 团队常用输入是否跑得通？ | 代表性的 prompt、源图、视频、音频或脚本能在选定 Card 或 Workshop 路径中完成 |
| 供应商访问是否清楚？ | Settings 中只配置了本次测试需要的直接供应商、中继或自定义中继路线 |
| 输出能否复用？ | 有价值结果被置顶或能通过 Asset Dock 和 SmartPicker 再次选择，而不是消失在一次性会话里 |
| Agent 是否有帮助且边界清楚？ | Agent 处理受控工作区动作，凭证、计费、删除和外部副作用仍由人审查 |
| 其他成员能否接手？ | Cards、Workshop 项目状态、置顶素材、导出结果和说明足够清晰，不依赖私人聊天上下文 |

## 45 分钟评估路径

| 步骤 | 动作 | 通过标准 |
| --- | --- | --- |
| 1. 选择一个真实场景 | 明确一个输入、一个输出和一个评审人 | 范围能在短会话内完成 |
| 2. 确认供应商路线 | 只配置该场景需要的模型家族 | Settings 中的直接供应商、中继或自定义中继与媒体类型匹配 |
| 3. 先跑手动基线 | 在使用 Agent 前先创建 Card 或 Workshop 项目 | 用户能检查 prompt、模型、状态、结果和错误 |
| 4. 复用一个输出 | 把结果保存到 Asset Dock，并通过 SmartPicker 再选一次 | 该素材可以作为后续参考 |
| 5. 尝试受控 Agent 协作 | 用明确目标请求 Agent 执行多步工作区任务 | 工具动作可审查，并产生可见工作区变化 |
| 6. 交接结果 | 下载输出、导出 Workshop 包，或记录所选素材和路线 | 其他成员不用猜隐藏状态即可继续 |

## 适配信号

| Redbit 更适合的情况 | Redbit 不太适合的情况 |
| --- | --- |
| 工作包含重复 prompt、参考素材、模型选择或评审步骤 | 只偶尔在单个供应商控制台生成一次 |
| 输出需要继续成为后续 Cards 或 Workshop 场景的输入 | 结果不需要复用、对比或项目结构 |
| 团队希望 BYOK，并显式控制供应商和中继路线 | 团队期待 Redbit 自带所有模型额度、配额或供应商 SLA |
| 执行者需要一个可见的 Agent 来调用受控工具 | 目标自动化需要无限制 shell 或未经审查的外部发布 |
| 创意、增长、支持和工程之间需要交接 | 工作始终只停留在一个人和一段聊天记录里 |

## 团队交接检查清单

| 交接物 | 为什么重要 |
| --- | --- |
| 选中的 Cards 和分组 | 展示 prompt、设置、状态、结果和失败 |
| 置顶 Asset Dock 素材 | 让下一位执行者能看到可复用输入 |
| Workshop 项目 | 保留脚本、场景、一致性参考、生成媒体和导出设置 |
| 供应商路线说明 | 说明结果来自直接供应商、中继、自定义中继还是 Local Core |
| 已知限制 | 记录视频慢、供应商错误、模型不一致或需要人工评审的地方 |
