# 核心工作流

Redbit 的工作流通常从一个想法、一份源素材或一段 brief 开始。之后每一步都会留在可检查的产品结构里：卡片负责单个任务，素材码头负责复用，工作坊负责项目结构，Agent 工具负责有边界的自动化。

![工作流图：灵感接入、卡片创建、生成、评审、交付、SmartPicker、工作坊和 Agent 协作。](../../assets/diagrams/redbit-creative-flow.svg)

## 用户角色

| 角色 | 在 Redbit 中通常做什么 |
| --- | --- |
| 独立创作者 | 生成图片、短视频、音频和 prompt 变体，同时保留可复用参考 |
| 创意运营 | 用系列图模板和工作坊场景构建产品、广告、角色或分镜资产 |
| 增长/CMO 运营 | 把创意素材、外部调研、本地自动化和可审查 Agent 动作组合起来 |
| 技术运营 | 配置供应商、中继、Local Core、MCP 挂载和 Agent 模型能力 profile |

## 从 Prompt 到卡片

<Steps>
  <Step title="选择模式">
    选择 `Image`、`Video`、`Speech`、`Audio` 或 `Describe`。所选模式决定可见模型、默认值、比例和输入要求。
  </Step>
  <Step title="准备 prompt 和参考">
    直接输入 prompt，添加源媒体，或通过素材码头/SmartPicker 使用已有结果。
  </Step>
  <Step title="生成并评审">
    运行单张卡片、选中卡片、分组或系列图。卡片上会显示状态、结果媒体、错误、收藏和下载入口。
  </Step>
  <Step title="复用结果">
    将输出保存到素材码头，作为后续参考，和其他素材对比，或把想法推进到工作坊。
  </Step>
</Steps>

## 从素材到新创意方向

素材复用是一条核心路径。上传图片、生成结果或文本 prompt 资产都可以成为：

- 图像生成的参考图；
- Seedance 视频的首帧、尾帧或参考素材；
- 工作坊中的场景图片；
- Agent 辅助改写的 prompt 来源；
- 用来对比、置顶或保留在项目中的素材。

SmartPicker 在没有项目上下文时打开全局素材码头；在工作坊项目上下文中可以切换到项目素材。它支持类型过滤、搜索、预览、单选和多选。

## 从脚本到视频包

工作坊适合管理多场景创意项目：

1. 从内置模板或空项目创建工作坊项目。
2. 编写或优化脚本。
3. 创建分镜场景，配置 prompt、旁白文本、时长和一致性参考。
4. 生成场景图片并选择源图。
5. 按需生成视频、旁白、音乐和节奏视频。
6. 通过导出面板导出项目资产和时间线元数据。

当前内置模板包括 15 秒产品广告、30 秒产品测评和空项目。代码中导出类型覆盖 FCPXML 和 EDL 相关结构，支持完整包和仅项目文件模式。

## Agent 协作工作流

当任务跨越多个 UI 动作时，Agent 更有价值：

- 创建、更新、复制、删除、选择、分组和生成卡片；
- 搜索或检查卡片/素材；
- 打开 SmartPicker；
- 切换标签或模块；
- 管理工作坊项目、场景、设置、一致性素材、媒体生成、导入/导出；
- 在配置可用时运行搜索、媒体、MCP、Local Core、CMO 或增长报告工具。

Agent 不是无限制 shell。注册工具元数据会标记读取、写入、破坏性、导航、工作流和外部副作用。破坏性项目/卡片操作需要明确目标或确认规则，工具结果也会在下一轮继续前归一化。
