# 核心概念与术语表

当产品、客户、支持或研发需要统一语言时，先看这一页。这里描述的是当前产品行为，不把路线图能力包装成已实现能力。

## 谁应该阅读本文

| 读者 | 适合用本文解决什么问题 |
| --- | --- |
| 新用户 | 在开始生成前，先理解工作区里的关键名称 |
| 客户沟通团队 | 解释 Redbit 时避免夸大模型归属、存储保证或 Agent 自主性 |
| 团队成员 | 统一产品文档、UI、支持话术和研发语境 |

## 前置概念

Redbit 是一个协调卡片、素材、供应商和可选本地集成的工作区。它本身不是模型供应商。生成质量、模型可用性、额度、地区访问和保留政策，取决于 Settings 中配置的直接供应商或中继。

## 核心术语

| 术语 | 含义 | 需要记住的边界 |
| --- | --- | --- |
| Cards | dashboard 中的独立任务单元，用于 `describe`、`image`、`video`、`audio` 和 `speech` | 卡片记录 prompt、模型设置、参考、状态和结果；实际供应商行为仍由所选模型决定 |
| Asset Dock | 本地可复用素材库，保存图片、视频、音频和文本 | 置顶素材用于长期复用；临时/最近素材可能被清理 |
| SmartPicker | 在全局素材或项目素材中选择资源的共用选择器 | 它帮助用户选择准确媒体，不会自行推断私有意图 |
| Workshop | 管理脚本、场景、一致性参考、生成媒体和导出的项目工作区 | 当任务变成多场景或项目结构时使用 |
| Agent runtime | 读取工作区上下文并调用已注册工具的受控工具循环 | 它不是无限制 shell；高风险或破坏性动作需要明确目标或确认策略 |
| Model Registry | 管理模型组、变体、比例、时长、输入要求、中继映射和供应商策略的目录 | 注册表描述应用支持情况，不保证供应商在线、可用额度或输出质量 |
| Provider | Settings 中配置的直接模型端点或兼容中继 | 用户负责凭证、计费、账号限制和供应商条款 |
| Reference | 被另一个任务作为输入或指导的既有图片、视频、音频或文本 | 可接受媒体和角色规则因模型而异 |
| Series | 图像卡片模式，从一个基础 prompt 规划 3、4、6 或 8 个相关槽位 | Series 适合做成组资产；每个槽位仍需要人工评审 |
| Seedance | 支持文生视频、图生视频、首尾帧和多参考模式的视频模型家族 | 参考角色比通用视频参考更严格 |
| Local Core | 可选本地引擎，支持部分存储、媒体、FFmpeg、MCP、自动化、插件和本地数据流程 | 只配对你自己启动并信任的本地引擎 |

## 术语使用约定

这些 UI 概念在中英文文档中保持稳定，不随意换名：

| 标准术语 | 用于 |
| --- | --- |
| Cards | dashboard 里的生成/评审任务单元 |
| Workshop | 项目级创意生产 |
| Agent runtime | 工具执行循环和助手模型 runtime profile |
| Model Registry | 基于目录的模型与供应商元数据 |
| Provider | 直接供应商或中继供应商 |
| Reference | 被其他任务使用的输入媒体/文本 |
| Series | 图像卡片的多槽位规划生成 |
| Seedance | Seedance 2.0 视频工作流 |
