# 卡片

卡片是 Redbit 的主要工作单元。一张卡片可以代表图像、视频、语音、音频或描述任务。每张卡片都保存自己的 prompt、模型设置、状态、结果列表、参考输入和可恢复的任务元数据。

## 谁应该阅读本文

当你需要理解生成任务会保留哪些信息、结果如何复用，以及 Series 或 Reference 什么时候会改变卡片行为时，读这一页。

## 常见操作

<Steps>
  <Step title="创建">
    使用 prompt 和可选模型设置添加一张或多张卡片。
  </Step>
  <Step title="生成">
    对待处理或已选择的卡片执行生成。
  </Step>
  <Step title="引用">
    将素材码头中的素材或其他卡片结果作为新卡片输入。
  </Step>
  <Step title="组织">
    随着工作区增长，你可以选择、分组、重排、删除、收藏、下载，或将输出置顶为素材。
  </Step>
</Steps>

## 卡片类型

| 类型 | 典型用途 |
| --- | --- |
| Describe | 将图片、音频或视频输入分析成可复用 prompt 文本 |
| Image | 生成单张图片或规划系列图 |
| Video | 基于 prompt、图片、帧、运动参考或模型专属输入生成视频 |
| Audio | 根据所选模型生成语音、音频或音乐相关输出 |
| Speech | 创建数字人/头像类语音视频输出 |

## 图像卡片的系列模式

图像卡片可以运行在 `single` 或 `series` 模式。系列模式会从一个基础 prompt 规划多个槽位，并跟踪每个槽位的 planned brief、可编辑 prompt、状态、选中输出和结果。

当前系列数量为 `3`、`4`、`6` 和 `8`。当前模板为 `auto`、`ecommerce`、`social`、`explainer`、`character`、`poster`、`brand` 和 `storyboard`。

## 参考素材

卡片可以使用上传文件、之前的卡片结果、素材码头或 SmartPicker 中的素材。参考行为取决于所选模型。例如，Seedance 使用明确的参考角色；其他视频模型可能根据注册表输入要求需要图片、运动视频或音频源。
