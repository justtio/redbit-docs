# 操作检查清单

在把真实任务交给 Redbit 前，先用这几张清单过一遍。每一项都对应一个会影响是否继续推进的实际判断。

## 谁应该阅读本文

| 读者 | 适用场景 |
| --- | --- |
| 工作区负责人 | 准备把 Redbit 环境交给用户使用 |
| 首次使用者 | 准备发起真实生成任务 |
| 支持/客户成功团队 | 视频或供应商工作流失败，需要做第一轮结构化排查 |

## 前置概念

先确认当前使用的是哪条路由：直接供应商、中继供应商、自定义中继，还是 Local Core。不要把凭证粘贴到 prompt、Cards、Workshop 脚本或 Agent 聊天里。

## 上线前配置

| 检查项 | 为什么重要 |
| --- | --- |
| 确认工作区入口和路径：`/dashboard`、`/workshop`、Settings | 用户需要稳定入口，才能开始模型配置 |
| 只配置当前工作流需要的供应商 key | 降低误暴露和误选供应商的风险 |
| 确认路由是直接供应商、中继还是自定义中继 | 可见模型和 payload 形状会随路由变化 |
| 如果要使用 Agent，先测试 describe/Agent runtime profile | tool calling、structured output、system prompt 和 vision 支持因模型而异 |
| 只有需要本地媒体、MCP、自动化、FFmpeg、插件或存储桥接时才配对 Local Core | 普通浏览器侧生成不需要本地引擎 |
| 上传机密素材前确认供应商条款 | Redbit 负责组织请求；隐私和保留政策仍由供应商决定 |
| 提前决定资产如何备份或导出 | 浏览器 profile 删除、站点数据清理或配额压力都可能移除本地数据 |

## 首次生成前检查

| 检查项 | 为什么重要 |
| --- | --- |
| 选择正确标签：`Describe`、`Image`、`Video`、`Audio` 或 `Speech` | 标签决定可见模型、默认值和输入要求 |
| 确认模型家族和变体 | 同一个 prompt 在不同模型上会产生不同结果 |
| 按需确认比例、时长、清晰度和音频设置 | 视频和图像供应商可能拒绝不支持的组合 |
| 只添加所选模型接受的参考素材 | Seedance、运动模型、图像编辑和 describe 模型的输入规则不同 |
| 除非任务需要 Agent 检查或路由素材，否则不要把源素材放进 Agent 聊天 | 避免素材进入不必要的助手上下文 |
| 准备复用的结果请保存或置顶 | 临时最近素材可能被清理 |
| 重试前先读卡片错误 | 不改凭证、输入或路由，通常只会重复同一失败 |

## 视频生成失败排查

| 现象 | 第一轮检查 |
| --- | --- |
| 缺少参考素材 | 确认模式和必需角色：首帧、尾帧、参考图、参考视频或参考音频 |
| Seedance 参考混用错误 | 同一个任务中只使用首尾帧角色，或只使用多参考角色，不要混用 |
| 供应商或中继报错 | 检查 API key 类型、base URL、中继能力、模型 ID 映射、额度和供应商状态 |
| 开启中继后模型消失 | 中继能力过滤可能不包含该模型家族或变体 |
| 视频很慢或卡住 | 尝试更短时长、更低清晰度、fast 变体，或换成队列压力更小的 provider 路由 |
| 结果和 prompt 不匹配 | 对比供应商/模型能力、参考强度、prompt 具体度，以及模型是否支持该输入类型 |
| Agent 说完成但没有媒体 | 检查生成工具是否真的返回 `{ success: true }`；只有文字回复不足以代表生成成功 |
